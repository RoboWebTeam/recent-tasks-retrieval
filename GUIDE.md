## Что это за проект

Roboweb — ИИ-сервис для создания и публикации сайтов на русском языке. Пользователь описывает идею словами в чате — сервис генерирует готовый одностраничный сайт (лендинг, магазин, визитку) через Claude Sonnet 5, позволяет его отредактировать в визуальном редакторе, опубликовать на своём поддомене или собственном домене и подключить приём заявок с сайта.

## Быстрый старт (разработка)

Проект состоит из трёх частей: React-фронтенд (Vite), Python cloud-функции backend и PostgreSQL + S3-хранилище.

### 1. Установка зависимостей фронтенда

```bash
npm install
# или, если используется bun
bun install
```

### 2. Запуск dev-сервера фронтенда

```bash
npm run dev
```

Откроется на `http://localhost:5173`. По умолчанию фронтенд обращается к уже развёрнутым облачным backend-функциям (адреса — в `src/lib/apiConfig.ts` и `backend/func2url.json`), поэтому для локальной разработки интерфейса отдельно поднимать backend не обязательно.

### 3. Backend

Backend не запускается локально одной командой — каждая папка в `/backend/*` является отдельной serverless-функцией на Python 3.11, которая деплоится изолированно. Структура одной функции:

```
backend/<имя-функции>/
  index.py          — точка входа: def handler(event, context) -> dict
  requirements.txt  — зависимости именно этой функции (если нужны сверх stdlib)
  tests.json        — обязательные тест-кейсы (метод, путь, ожидаемый статус)
```

Изменения в backend применяются деплоем конкретной изменённой функции — в среде разработки poehali.dev это делает инструмент синхронизации бэкенда автоматически после правки файлов и прогоняет тесты из `tests.json`.

### 4. База данных

PostgreSQL. Схема управляется миграциями в `db_migrations/` (файлы вида `V0001__create_leads_table.sql`, `V0002__create_users_sessions_projects.sql` и т.д., применяются строго по порядку номеров, отката назад нет). **Никогда не редактируйте уже применённую миграцию** — любое изменение схемы оформляется новым файлом с очередным номером.

### 5. Сборка продакшн-версии фронтенда

```bash
npm run build
```

Результат — статические файлы в папке `dist/`.

## Переменные окружения (секреты)

Все секреты хранятся в защищённом хранилище секретов проекта (никогда не в файле `.env` внутри репозитория) и доступны backend-функциям через `os.environ`. Шаблон полного списка — `deploy/.env.example`. Ключевые секреты:

| Секрет | Назначение |
|---|---|
| `DATABASE_URL` | Строка подключения к PostgreSQL |
| `MAIN_DB_SCHEMA` | Имя используемой схемы БД |
| `AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY` | Ключи встроенного S3 платформы (легаси, старые файлы блога/портфолио) |
| `REG_S3_ACCESS_KEY_ID` / `REG_S3_SECRET_ACCESS_KEY` | Ключи собственного S3-хранилища на reg.ru (Рег.облако, bucket `roboweb`) — используется для ВСЕХ новых загрузок пользователей |
| `ANTHROPIC_API_KEY` | Ключ Anthropic API — прямая генерация HTML сайтов через Claude Sonnet 5 |
| `FUNCTION_TIMEOUT_SEC` | Таймаут функции `generate-site` в секундах (обязан совпадать с настройками самой функции; 90 сек нужно для качественной генерации) |
| `GITHUB_CLIENT_ID` / `GITHUB_CLIENT_SECRET` | OAuth-приложение GitHub (вход + экспорт сайта на GitHub Pages) |
| `YANDEX_CLIENT_ID` / `YANDEX_CLIENT_SECRET` | OAuth-приложение Яндекс ID |
| `YOOKASSA_SHOP_ID` / `YOOKASSA_SECRET_KEY` | Приём онлайн-платежей ЮKassa |
| `SMTP_PASSWORD` | Отправка писем с почты `roboweb.site@yandex.ru` через smtp.yandex.ru |
| `ADMIN_KEY` | Пароль входа в админ-панель `/admin` |

## Структура репозитория

```
src/                     Frontend (React + TypeScript + Vite + Tailwind CSS)
  pages/                 Страницы приложения (все маршруты — в src/App.tsx)
    admin/               Компоненты и вкладки админ-панели
    index/               Секции и данные главной страницы
    legal/               Юридические страницы (оферта, приватность)
  components/
    ui/                  Базовые shadcn/ui компоненты (кнопки, диалоги, формы...)
    builder/             Компоненты редактора сайтов (панель ядра, домен-модалка...)
    dashboard/           Вкладки личного кабинета (проекты, тариф, профиль)
    domain/              Управление подключёнными доменами
    demo/                Демо-режим без регистрации
    extensions/yookassa/ Кнопка оплаты
  lib/
    auth.ts              Единый API-клиент (все функции apiXxx для вызова backend)
    apiConfig.ts         Таблица соответствия имя-функции → URL
    i18n.ts              Переводы RU/EN
    seo.ts               Установка meta-тегов и JSON-LD
    analytics.ts         Отправка целей в Яндекс.Метрику
  data/                  Статические данные (статьи блога, примеры портфолио)

backend/                 Backend (Python 3.11, каждая папка — отдельная cloud-функция)
  <function-name>/
    index.py             Обработчик: def handler(event, context) -> dict
    tests.json           Обязательные тесты функции
  func2url.json           Автогенерируемое соответствие функция → публичный URL

db_migrations/           SQL-миграции PostgreSQL (применяются по порядку номеров)

deploy/                  Перенос проекта на собственный VPS
  README.md              Пошаговый гайд (ручная установка или Docker Compose)
  docker-compose.yml, Dockerfile, frontend.Dockerfile, nginx.conf, server.py
  .env.example           Шаблон всех переменных окружения
  export_db.sh           Скрипт экспорта текущей БД в дамп

ARCHITECTURE.md          Полное описание логики и функционала проекта
TECH_STACK.md            Технологический стек и список внешних интеграций
GUIDE.md                 Этот файл
```

## Как вносить изменения

1. **Frontend** — правьте файлы в `src/`. Строго соблюдайте существующую структуру: базовые визуальные элементы — только через `src/components/ui`, не создавайте им дублирующие аналоги. Для иконок использовать только компонент `<Icon name="..." />` из `src/components/ui/icon.tsx` (обёртка над lucide-react), никогда не импортировать иконки напрямую.
2. **Backend** — правьте `index.py` нужной функции. После любых изменений функция обязана проходить собственные тесты из `tests.json`. Функции полностью независимы друг от друга — деплоятся и тестируются по отдельности, не меняйте несколько функций одновременно без необходимости.
3. **База данных** — любое изменение схемы (новая таблица, колонка, индекс) только через новый файл миграции в `db_migrations/`. Никогда не выполняйте `ALTER`/`CREATE` в обход системы миграций — тогда локальная копия и продакшн разойдутся.
4. **Секреты** — никогда не хардкодьте ключи и пароли в коде. Новый секрет запрашивается через механизм секретов проекта и читается в backend строго через `os.environ['ИМЯ_СЕКРЕТА']`.
5. **CORS** — каждая backend-функция обязана сама обрабатывать `OPTIONS`-запрос (preflight) и возвращать заголовок `Access-Control-Allow-Origin: *` во ВСЕХ ответах, включая ошибки — иначе фронтенд не сможет получить даже текст ошибки.

## Перенос на собственный сервер (VPS)

Полная пошаговая инструкция (PostgreSQL, nginx, systemd, Docker Compose, перенос базы, файловое хранилище, OAuth callback-адреса) — см. **[deploy/README.md](./deploy/README.md)**.

Коротко: один и тот же билд фронтенда не может одновременно работать и на облаке платформы, и на собственном VPS — при сборке для VPS нужно явно указать `VITE_API_BASE_URL=https://ваш-домен npm run build`, тогда все запросы пойдут через `/api/<имя-функции>` на ваш сервер вместо облачных функций.

## Домены и публикация сайтов пользователей

- Каждый сгенерированный сайт можно опубликовать бесплатно на поддомене вида `roboweb.site/site/<slug>` — без дополнительной настройки.
- Подключение собственного домена клиента — раздел «Домен» в редакторе или страница `/settings/domain`, с автоматической проверкой DNS (A/CNAME записи через Google DNS-over-HTTPS) и последующим выпуском SSL.
- Экспорт HTML-кода сайта на GitHub Pages — кнопка «GitHub» в редакторе (требует предварительной привязки GitHub-аккаунта через OAuth).

## Диагностика проблем

- **Frontend падает / белый экран** — сначала смотрите консоль браузера (там же видны ошибки сети к backend).
- **Backend возвращает 500** — смотрите логи конкретной функции; почти всегда причина — необработанное исключение или отсутствующий секрет в `os.environ`.
- **Генерация сайта не работает** — проверьте, что секрет `ANTHROPIC_API_KEY` задан и что `FUNCTION_TIMEOUT_SEC` совпадает с реальным таймаутом функции `generate-site` в настройках (Sonnet 5 требует минимум 60–90 секунд на сложный сайт).
- **Файлы не загружаются** — проверьте секреты `REG_S3_ACCESS_KEY_ID` / `REG_S3_SECRET_ACCESS_KEY` и что объекты в bucket `roboweb` создаются с ACL `public-read` (иначе reg.ru отдаёт 403 на публичной ссылке).
